# 品牌命名策划

目标：为下一阶段的新仓库 / 新品牌提供一组更适合长期演进的名字。产品未来不应被限制为聊天客户端，而应能覆盖：

- workflow / agent
- desktop assistant
- CLI
- IM / 外部接入
- 工具编排与执行系统

命名标准：

- 短
- 有力量感
- 中英文都能成立
- 不锁死 `chat` / `desk`
- 能承载“执行、调度、中枢、工作流”语义
- 尽量避免过于通用或强冲突的大词

## 结论先看

第一梯队：

1. `枢流 / Shuflow`
2. `流枢 / Fluxis`
3. `核流 / Coreflow`
4. `元流 / Metaflow`

第二梯队：

5. `宙流 / Omniflow`
6. `脉核 / Pulsecore`
7. `引流 / Guidra`
8. `流核 / Flowcore`

第三梯队：

9. `极流 / Fluxa`
10. `曜核 / Velcore`
11. `元枢 / Metahub`
12. `执流 / Execflow`

## 一打候选

### 1. 枢流 / Shuflow

定位感：
AI 工作流中枢 / 调度核心

优点：
- 中文有辨识度
- `枢` 很像“中枢、控制台、核心调度”
- `流` 能覆盖 workflow / message flow / task flow
- 不限制桌面，也不限制聊天

风险：
- `Shuflow` 需要稍微教育读音

适合度：
很高

### 2. 流枢 / Fluxis

定位感：
偏系统、偏品牌、偏国际化

优点：
- 比 `FlowX` 更像正式品牌
- 有“流动 + 中枢 + 执行系统”的味道
- 英文比纯拼音更国际化

风险：
- 中文和英文不是直译关系，品牌解释需要统一

适合度：
高

### 3. 核流 / Coreflow

定位感：
执行核心 / workflow kernel

优点：
- 很贴你现在的产品内核
- 容易延展出 `Coreflow CLI`、`Coreflow Agent`
- 技术气质强

风险：
- 英文略偏通用，品牌独占性一般

适合度：
高

### 4. 元流 / Metaflow

定位感：
更像平台层、框架层、系统层

优点：
- 有平台感
- 适合未来多入口
- 中文比较稳

风险：
- `Metaflow` 这个英文在技术圈已有较多使用，独占性一般

适合度：
中高

### 5. 宙流 / Omniflow

定位感：
全域、多入口、多场景 AI 流系统

优点：
- 适合你未来做桌面、CLI、IM、agent runtime
- 品牌气势大

风险：
- 范围感太大，容易显得有点“铺太开”

适合度：
中高

### 6. 脉核 / Pulsecore

定位感：
活跃、执行、持续运行的智能核心

优点：
- 有生命力
- 技术感不错
- 比较适合 agent 系统

风险：
- 和 workflow 的直接关联不如 `枢流`

适合度：
中高

### 7. 引流 / Guidra

定位感：
引导执行、辅助推进、智能导航

优点：
- 有“pilot”那种引导感
- 适合辅助执行系统

风险：
- `引流` 中文容易联想到营销语境

适合度：
中

### 8. 流核 / Flowcore

定位感：
workflow engine / core runtime

优点：
- 很直给
- 适合作为技术品牌或仓库名

风险：
- 品牌味弱，更像技术模块名

适合度：
中高

### 9. 极流 / Fluxa

定位感：
短、狠、偏国际化

优点：
- 很像现代软件品牌
- 英文短，适合域名、包名、CLI

风险：
- 中文含义不如 `枢流` 稳
- 需要靠视觉和文案补世界观

适合度：
中

### 10. 曜核 / Velcore

定位感：
高性能、高密度、核心系统

优点：
- 品牌感强
- 中文气质不错

风险：
- 和 workflow / assistant 的关联较弱

适合度：
中

### 11. 元枢 / Metahub

定位感：
多智能体与多工作流的中心节点

优点：
- 很像平台中台
- 中文不错

风险：
- `hub` 太常见
- 稍偏平台，不够锋利

适合度：
中

### 12. 执流 / Execflow

定位感：
从思考到执行的流程系统

优点：
- 很贴你的真实产品方向
- 不是聊天，而是执行

风险：
- `Execflow` 稍硬，商业品牌感一般

适合度：
中高

## 不建议继续深挖的方向

### 1. 带 `Chat`

问题：
- 继续把产品锁死为聊天软件
- 和未来 workflow / agent / automation 冲突

### 2. 带 `Desk`

问题：
- 限定桌面
- 后续 CLI / IM / API 接入会别扭

### 3. `OpenFlow`

问题：
- 已被 SDN / 网络技术标准强占
- 不是可控品牌位

### 4. `AgentX`

问题：
- 太泛
- 现成项目与协议占用太多

### 5. `FlowX`

问题：
- 风格上行，但太像泛科技名字
- 品牌独占性不够稳

## 推荐策略

### 方案 A：中文主气质 + 英文国际化落地

推荐：

- `枢流 / Shuflow`
- `脉核 / Pulsecore`
- `宙流 / Omniflow`

适合：
- 你想保留中文辨识度
- 又不想英文太土

### 方案 B：直接走国际化主品牌

推荐：

- `Fluxa`
- `Flowcore`
- `Coreflow`

适合：
- 新仓库、新域名、新 CLI 一套统一

### 方案 C：更偏平台 / 系统感

推荐：

- `Shuflow`
- `Coreflow`
- `Metahub`

适合：
- 未来想做 agent runtime / workflow system / desktop assistant platform

## 当前建议

如果只选 3 个，我建议你重点考察：

1. `枢流 / Shuflow`
2. `核流 / Coreflow`
3. `极流 / Fluxa`

建议理由：

- `Shuflow`：最像你自己的东西，有中文气质，也能承载未来
- `Coreflow`：最稳，最贴当前产品结构
- `Fluxa`：最像现代新品牌，短、利落、好包装

## 下一步建议

筛选流程建议：

1. 从这 12 个里先砍到 5 个
2. 再做一轮：
   - GitHub 仓库名可用性
   - 域名可用性
   - 包名 / CLI 名称顺手程度
   - 中文读感
   - 英文读感
3. 最后在剩下 2-3 个里做视觉稿和 README 首页样机
