# 阶段9测试验收文档

> 测试目的：验证任务账本与高级 UI 仅消费结构化状态，不反向污染内核  
> 对应计划：`work/plan.md` 第 16 节和 16B 节

---

## 一、阶段测试边界

本阶段只做展示层增强与结构化任务态接入。

- 必测：TaskItem/Inspector/高级面板数据来源、前端异常隔离
- 不测：执行内核状态机改造（应保持稳定）

---

## 二、核心验证点

1. UI 数据来源为结构化状态或事件
2. 不从 transcript 文本推断任务状态
3. UI 异常不影响 workflow 执行与恢复
4. 多会话切换下任务面板一致
5. 焦点代理/任务账本展示一致，不篡改执行状态

---

## 三、日志与观测建议

建议增加或关注：

```log
workflow.ui.task_ledger.render
workflow.ui.task_ledger.data_source
workflow.ui.task_ledger.error
```

并确认出现 UI 错误时，后端执行日志仍正常推进。

---

## 四、自动化测试最低要求

1. TaskItem 结构映射测试
2. 前端选择会话/焦点切换的状态一致性测试
3. UI 渲染异常时执行链不受影响（集成或 e2e）
4. 禁止 transcript 推断路径的测试（如 lint/静态规则或单测）

---

## 五、测试场景（自动化优先）

### 场景1：结构化任务列表渲染（自动化）

建议实现：
1. 前端单测构造结构化任务态 store
2. 渲染 TaskItem/Inspector
3. 断言字段映射与展示一致

验收：
- ✅ 任务列表完整
- ✅ 字段与后端结构一致

### 场景2：不依赖 transcript 推断（自动化）

建议实现：
1. 注入噪声 transcript + 正确结构化状态
2. 断言任务态仅来自结构化字段
3. 增加静态规则或单测阻止 transcript 推断路径

验收：
- ✅ 任务状态不受 transcript 文本污染
- ✅ UI 仍以结构化数据为准

### 场景3：会话切换一致性（自动化 + 手工补充）

自动化建议：
1. store 单测模拟多 session 高频切换
2. 断言任务账本和焦点状态不串台

手工补充：
1. UI 快速切换多个会话
2. 观察是否出现短暂残影/串闪

验收：
- ✅ 不串 session
- ✅ 不出现历史残留状态

### 场景4：UI 异常隔离（自动化 + 手工补充）

自动化建议：
1. 前端组件测试中注入 render error
2. 断言错误被边界捕获/降级
3. 后端集成测试断言 workflow 继续推进

手工补充：
1. 开启真实 UI 后触发异常面板
2. 验证用户可见提示与回退体验

验收：
- ✅ 前端可提示错误或降级
- ✅ workflow 执行与恢复不受影响

### 场景5：重启后展示恢复（自动化 + 手工补充）

自动化建议：
1. 构造 snapshot/event 后重建前端初始态
2. 断言任务面板与后端状态一致

手工补充：
1. 真机重启后观察首屏恢复过程
2. 验证加载态与最终态切换无明显误导

验收：
- ✅ 展示可恢复
- ✅ 与 snapshot/event 状态一致
