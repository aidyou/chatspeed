# Chatspeed Bug 修复与功能优化计划

## 一、 数据库与基础数据模型重构
- [ ] **数据库迁移 (v5.rs):**
    - 修改 `src-tauri/src/db/sql/migrations/v5.rs` 中的 `workflow` 表结构。
    - **删除字段:** `allowed_paths`, `final_audit`。
    - **新增字段:** `agent_config` (TEXT 类型，存储 JSON 化的智能体配置)。
- [ ] **Rust 结构体清理:**
    - 修改 `src-tauri/src/db/types.rs`（或相关文件）中的 `Workflow` 结构体，映射新的字段。
    - 修改 `Agent` 结构体或相关的 `ModelConfig`：**删除** `coding_model`, `writing_model`, `browsing_model` 及其相关处理逻辑。
- [ ] **DAO 层适配:**
    - 更新 `src-tauri/src/db/workflow.rs`，支持 `agent_config` 的序列化与反序列化。
    - 确保 `WorkflowExecutor` 在初始化时从 `agent_config` 中读取 `shell_policy`, `auto_approve` 等配置。

## 二、 前端界面优化与修复 (`src/views/Workflow.vue` & `Agent.vue`)
- [ ] **智能体设置 (`Agent.vue`):**
    - 移除“代码模型”、“写作模型”、“浏览模型”的配置项。
- [ ] **自动审批列表管理:**
    - 当选择“根据智能体配置”模式时，展示当前已自动审批的工具列表和 Shell 规则列表。
    - 支持用户从列表中直接删除特定工具或 Shell 规则。
    - **同步机制:** 删除后立即更新数据库（`agent` 表或 `workflow` 的 `agent_config`）以及当前运行中的 `WorkflowExecutor` 实例。
- [ ] **工具调用展示增强:**
    - **Diff 展示:** 针对 `edit_file` 和 `write_file`，在审核弹窗和消息列表中实现类似于代码差异对比的展示效果。
    - **命令可读性:** 优化工具调用详情展示，例如 `shell_execute` 应直接突出显示 `command` 内容，方便用户快速审查。
- [ ] **状态与交互修复:**
    - **审批弹窗失效修复:** 解决页面切换后“No sender for session”导致的审批无法操作问题。需检查 `Gateway` 在页面重新挂载时的连接恢复逻辑。
    - **模型配置即时生效:** 修复 `/models` 命令修改后前端状态未刷新的问题，确保 Pinia store 或本地状态同步更新。
    - **I18N 更新:** 将“智能审批 (只拦截写操作)”改为“智能审批 (只拦截风险操作)”。

## 三、 后端流程与逻辑优化
- [ ] **Shell 安全策略增强:**
    - 确保后端正确应用“智能审批”设置，对于 `git log` / `diff` 等已在白名单的命令不触发拦截。
    - **自动审批规则泛化:** 实现“全部同意”时的规则自动生成（例如 `cat /a/b/c` 同意后，生成 `cat *` 或 `cat /a/b/*` 规则），并持久化到 `workflow.agent_config.shell_policy`。
- [ ] **工作流引擎改进:**
    - **异步标题生成:** 将工作流标题生成逻辑改为异步任务（Spawn），避免阻塞主执行流程。
    - **步数耗尽处理:** 步数耗尽后不再强制停止，而是调用 `ask_user` 询问用户是否继续。若用户同意，后端重置步数并切换回 `Thinking` 状态。
    - **异常处理降级:** 将“操作已被用户取消”从 `Critical Error` 降级为普通状态变更。确保用户取消后，流程处于“已暂停”状态，允许用户重新发起指令或调整后再继续，而不是直接中断整个工作流引擎。
    - **压缩逻辑修复:** 解决上下文压缩后立即再次触发压缩的死循环问题（检查触发阈值与压缩后的 token 计算逻辑）。
- [ ] **通知与反馈优化:**
    - **Shell 结果即时通知:** 研究并实现将 Shell 运行的实时输出（stdout/stderr）通过 `Gateway` 即时推送到前端展示。
    - **自动审批透明化:** 即使是自动审批的命令，也必须将执行详情发送给前端，确保用户知晓后台操作，并修复工具状态始终处于“执行中”的 Bug。

## 四、 实施顺序
1.  **后端基础重构:** 先清理模型定义并完成数据库迁移。
2.  **前端 UI 调整:** 匹配新的模型配置并优化消息展示样式。
3.  **核心逻辑修复:** 重点解决审批失效、Shell 策略和步数耗尽流程。
4.  **功能完善:** 标题异步化、Shell 结果流式通知等增强功能。
